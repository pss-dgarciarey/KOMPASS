# Contribution Policy

Kompass is a source-available, curator-led project.

Public discussion is welcome, but public code contributions are not
automatically accepted.

That is deliberate. The repo is public for transparency and review, not as a
free-for-all code commons.

## What is welcome

- bug reports with reproducible steps
- factual corrections
- source suggestions
- UI or methodology feedback

## What requires prior written permission

- pull requests intended for merge
- forks used beyond private evaluation
- derivative deployments
- redistribution of the codebase
- organizational or commercial use

## If you want to propose a change

1. Describe the issue clearly.
2. Include screenshots, logs, or payloads when relevant.
3. Explain the expected impact on signal quality, cost, or reliability.
4. Wait for maintainer approval before preparing a code contribution.

Short version: start with the problem, not with a patch.
